CS499 Computer Science Capstone, Dec 2024

During my time in the Southern New Hampshire University Computer Science program, as well as my career in data analytics, I have developed and honed a wide range of skills related to the field of Computer Science. Through the academic program I gained many valuable technical skills, including experience with popular programming languages including Java, C++, and Python. I learned to effectively leverage both SQL and NoSQL databases. I gained familiarity with popular frameworks like Angular, Express and Node.js.

While these technical skills are important, they represent only the foundation for delivering value using Computer Science expertise. The program also exposed me to common project management approaches used in the software development lifecycle and emphasized the importance of thoughtful requirements gathering and consideration for the needs of stakeholders. Effective communication with team members, project managers, product owners, customers, and other stakeholders was a critical element in the program. I also learned to weigh ethical considerations as part of my decision-making process and in the implementation of new projects, aligning with professional ethical guidelines such as the Association for Computing Machinery (ACM) Code of Ethics and Professional Conduct.

As the effective application of Computer Science often involves working with sensitive personal or proprietary information, the importance of strong security practices, as well as adherence to relevant regulatory requirements, was also a point of emphasis. Throughout my time in the Computer Science program, I have internalized the importance of implementing security best practices early in the development process and at every level of a program.

To showcase my evolving skills in a variety of Computer Science applications, I will present three artifacts from different courses in the program. Broadly, these artifacts will align with the following categories:

  • Software Design and Engineering

For each of these artifacts, I will present an original artifact from earlier in the program. I will perform a video code review of that artifact, discussing its strengths and weaknesses, then I will propose and implement enhancements for that artifact, improving its functionality and aligning it better with professional best practices. Together, these artifacts demonstrate my skill with popular programming languages C++, Java, and Python, my facility with SQL and NoSQL databases, my focus on robust application security, and my ability to add value to end-user workflows.

Software Design and Engineering Artifact

Original Artifact

The Software Design and Engineering artifact that I am demonstrating is an Android mobile application for tracking inventory items. The application supports allowing users to create accounts and log in, add items to a database, edit the quantity of items that currently exist in the database, and remove items from the database. The application also optionally supports SMS notifications when stock levels are low. I originally created this artifact for CS360: Mobile Architecture and Programming in August 2024.

This artifact is an excellent example of my skills in software development and design for several reasons. First, the application supports an end-to-end workflow for a use case that is common across many industries- tracking stock of items- which makes it an ideal example of the value I can add to an organizationâ€™s business cases through software development.

Additionally, because the Android platform is popular across many devices, architectures, and applications, it demonstrates my ability to develop applications with broad versatility to meet many possible user requirements. The application is specifically designed to enable collaboration between multiple users as they track and update the status of a single inventory, enabling accurate and current information to be available to stakeholders.

Enhanced Artifact

Information security is a critical concern to nearly every industry that relies on computing solutions. To reflect this, I chose to enhance this artifact by adding additional security features to both better align with best practices and allow organizations more precise control over how their data is modified. To achieve these goals, I implemented password hashing to protect user credentials from compromise, applied role-based access controls to user accounts, and implemented the principle of least privilege to critical database functionality such as creating or deleting records. Administrators now have the ability to set the access level of users, restricting some functions only to users who directly require them.
